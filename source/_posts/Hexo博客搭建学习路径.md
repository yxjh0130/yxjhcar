## 🎯 学习路径总览：从零搭建 Hexo + AnZhiYu 博客

你不需要先学网页开发再搭建网站。**Hexo 的设计理念就是让不懂代码的人也能搭建博客**。建议你按以下顺序边学边做：

---

### 第一阶段：理解你手里的东西（1-2天）

在开始操作之前，先搞清楚几个核心概念：

| 概念 | 大白话解释 |
|------|-----------|
| **Hexo** | 一个"文章转网站"的工具。你写 Markdown 文章，它把你写的文章变成漂亮的网页 |
| **Markdown** | 一种极简的写作格式，用 `#` 表示标题，`**文字**` 表示加粗，比 Word 简单一万倍 |
| **Node.js** | Hexo 运行需要的环境，你电脑上已经安装了（否则 node_modules 文件夹不会存在） |
| **主题 (Theme)** | 网站的"皮肤"。AnZhiYu 就是你选的皮肤，决定了网站长什么样 |
| **`_config.yml`** | 配置文件。用来调整网站外观和功能的 |
| **静态网站** | 最终生成的就是一堆 `.html` 文件，不需要数据库，上传到任何地方都能访问 |

**✅ 动手任务：**

1. 在当前项目目录打开终端，运行 `npm run server`，然后在浏览器打开 `http://localhost:4000`，看看你的网站长什么样
2. 浏览 `themes/anzhiyu/_config.yml`，对照注释理解每个配置项

---

### 第二阶段：学会写文章（1天）

这是最核心的技能，也是最简单的。

**Markdown 基础语法（只需10分钟学会）：**

```markdown
# 一级标题
## 二级标题
### 三级标题

**加粗文字**
*斜体文字*

- 列表项1
- 列表项2

1. 有序列表1
2. 有序列表2

[链接文字](https://网址)
![图片描述](图片地址)

> 引用文字

`行内代码`

​```语言
代码块
​```
```

**✅ 动手任务：**

1. 运行 `npx hexo new "我的第一篇文章"` 创建一篇文章
2. 用 VS Code 打开 `source/_posts/我的第一篇文章.md`，写一段话试试
3. 运行 `npm run server` 预览效果

---

### 第三阶段：配置你的网站（2-3天）

修改 `_config.yml` 配置文件。需要理解 **哪些配置是必须改的**：

#### 🔴 必须修改的（不改网站跑不起来）：

| 配置项 | 位置 | 说明 |
|--------|------|------|
| `title` | 根目录 `_config.yml` | 网站标题，改成你自己的 |
| `subtitle` | 根目录 `_config.yml` | 副标题 |
| `author` | 根目录 `_config.yml` | 作者名 |
| `url` | 根目录 `_config.yml` | 网站网址，部署前必须改 |

#### 🟡 强烈建议修改的：

| 配置项 | 位置 | 说明 |
|--------|------|------|
| 头像/logo | 主题 `_config.yml` 的 `avatar` | 你的头像图片 |
| 导航菜单 | 主题 `_config.yml` 的 `menu` | 顶部或侧边栏菜单 |
| 页脚信息 | 主题 `_config.yml` 的 `footer` | 底部版权信息 |
| 社交链接 | 主题 `_config.yml` 的 `social` | GitHub、QQ 等 |

#### 🟢 锦上添花的（可以慢慢调）：

- 评论系统（需要注册第三方服务）
- 访问统计（百度统计/Google Analytics）
- 背景特效（彩带、粒子效果）
- 音乐播放器

---

### 第四阶段：部署上线（1天）

网站写好了，需要让别人能访问。对于新手，**最推荐 GitHub Pages（完全免费）**：

**简化流程：**

1. **注册 GitHub 账号**（github.com，免费）
2. **创建一个仓库**，名字必须叫 `你的用户名.github.io`
3. **安装 Git**（如果你电脑上没有的话）
4. **配置部署**：在根目录 `_config.yml` 中找到 `deploy` 部分，填上你的仓库地址
5. **运行部署命令**：`npm run deploy`

部署后你的网站就上线了，任何人都能通过 `https://你的用户名.github.io` 访问。

---

### 第五阶段：进阶学习（可选，按需学习）

当你熟悉基本操作后，如果还想深入：

| 你想做的事 | 需要学什么 | 难度 |
|-----------|-----------|------|
| 调整页面布局/颜色 | CSS 基础 | ⭐⭐ |
| 添加自定义交互 | JavaScript 基础 | ⭐⭐⭐ |
| 修改主题模板 | EJS/Pug 模板语法 | ⭐⭐⭐ |
| 买自己的域名 | DNS 解析配置 | ⭐ |
| 用 VPS 服务器部署 | Linux 基础命令 | ⭐⭐⭐ |

## 📋 你的行动清单（按优先级排序）

- [ ] 1. 运行 `npm run server`，在浏览器预览你的网站
- [ ] 2. 修改根目录 `_config.yml` 中的 `title`、`subtitle`、`author`
- [ ] 3. 写一篇测试文章，预览效果
- [ ] 4. 修改主题 `_config.yml` 中的头像、菜单、社交链接
- [ ] 5. 注册 GitHub 账号
- [ ] 6. 安装 Git
- [ ] 7. 配置 `deploy` 部署信息
- [ ] 8. 运行 `npm run deploy` 上线
- [ ] 9. 后续持续写文章 + 调整配置

---

## ⚠️ 给零基础用户的特别提醒

1. **不要被代码吓到** — 你只需要改配置文件，不需要写代码
2. **遇到报错先看错误信息** — 大部分报错信息会告诉你哪里出错了
3. **配置文件缩进很重要** — YAML 格式用空格缩进，不要用 Tab，缩进错了网站就崩
4. **改配置前先备份** — 复制一份原文件，改坏了可以恢复
5. **每次改完配置就 `npm run server` 预览** — 确认没问题再继续改下一个
